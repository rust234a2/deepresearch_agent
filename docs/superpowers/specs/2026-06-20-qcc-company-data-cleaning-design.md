# 企查查企业工商数据清洗设计

## 目标

将企查查批量查询 Excel 清洗为可用于后续供应商资料补充的结构化 CSV。清洗过程可重复执行，原始文件和派生数据均保留在本地且不提交 Git。

## 输入规则

- 第一行是数据使用声明，跳过。
- 第二行是表头。
- 第三行开始是企业数据。
- `-`、`--`、空字符串和“暂无”统一视为空值。
- “系统匹配企业名称”为“未匹配到相关企业”的记录进入拒绝表。

## 输出

输出目录：`data/procurement/cleaned/`

- `companies.csv`：核心企业工商数据。
- `contacts.csv`：电话、邮箱和通信地址。
- `rejected.csv`：无法匹配或缺少核心身份字段的记录及原因。

## 核心企业字段

- 来源名称、法定企业名称、登记状态、统一社会信用代码。
- 法定代表人、企业类型、成立日期、营业期限。
- 注册资本与实缴资本：保留原始文本，同时输出基础货币单位金额和币种。
- 企业地址、省、市、区县、登记机关。
- 国标行业门类、大类、中类、小类。
- 企业规模、经营范围、曾用名、英文名、官网。
- 参保人数、参保人数年报、最新年报年份、纳税人资质。

## 联系方式字段

- 统一社会信用代码、法定企业名称。
- 电话列表、邮箱列表、通信地址。
- 多值字段使用 `|` 分隔并去重。

## 清洗规则

- 企业名称去除首尾空白和异常换行。
- 曾用名、电话、邮箱拆分、去重后使用 `|` 连接。
- 资本中的 `万` 乘以 10000；人民币、美元、港元分别规范为 `CNY`、`USD`、`HKD`。
- 日期统一为 `YYYY-MM-DD`。
- “无固定期限”拆为结束日期为空、无固定期限标记为 `true`。
- 参保人数和年报年份只保留数字。
- `企业简介` 不进入结果，因为导出内容明确包含 AI 生成文本。
- 纳税人识别号、注册号、组织机构代码不进入核心表，避免与统一社会信用代码重复。

## 数据治理

- 原始 Excel 和清洗后的 CSV 加入 Git 忽略规则。
- 清洗脚本、测试和字段定义可以提交 Git。
- 清洗结果只用于本地项目，不向第三方分发。

## 验收标准

- 3509 条输入应得到 3506 条核心企业记录和 3 条拒绝记录。
- 核心表中法定企业名称和统一社会信用代码不得为空。
- 三张 CSV 均为 UTF-8 with BOM，可直接使用 Excel 打开。
- 资本、日期、多值字段和缺失值转换均有自动化测试。
